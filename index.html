What does "Powered by BNRP" mean?

BNRP stands for Bitcoin Name Resolution Protocol. BTCNative uses BNRP open resolution to help verify and display BTC-native name records, including owner data, address records, avatar or inscription links, primary identity records, and marketplace listing status.

BNRP resolves
  • name → owner address
  • name → primary Bitcoin address
  • name → avatar / inscription
  • name → text records
  • name → marketplace listing status
  • address → primary name (reverse lookup)
  • namespace → supported resolver / indexer
  • subname → parent namespace + authorization
  • record → history / current state

BNRP does not own the names. It is the resolver layer that helps BTCNative read and organize name data in a consistent way.

Permanent history. Current state. Bitcoin inscriptions are permanent. BNRP reads the latest valid inscription — not just the newest. A valid update must be authorized by the current owner or delegate, match namespace policy, and not be revoked.

Permanent history.
Current state.

Bitcoin inscriptions are permanent. BNRP does not edit old inscriptions. Instead, each update creates a new Bitcoin-native record. The resolver reads the latest valid record, while the full history remains visible.

Latest valid inscription wins.

A newer inscription only becomes the active record if it is valid. BNRP checks whether the update was authorized by the current owner or delegated manager, whether it follows the namespace policy, whether it uses a newer nonce or version, and whether it has not been revoked or superseded.
